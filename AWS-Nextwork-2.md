# Cloud Security with AWS IAM
**By Vhon Daniel Pimentel** | July 2026

## 🚀 Project Overview
For this project, I got to dive into AWS by launching and managing EC2 instances for scalable computing and getting hands-on with IAM to set up secure access controls with Policies, Users, and User Groups. The coolest part was seeing how this actually applies to the real world, like using Account Aliases to customize sign-in URLs and automating permissions for new hires, which I got to test out firsthand by logging into my own AWS account as a new user.

---

## 🛠️ Tools and Concepts
* **Services Used:** AWS Identity and Access Management (IAM), Amazon Elastic Compute Cloud (EC2)
* **Key Concepts Learned:** 
  * User management and access control
  * Deploying virtual machine instances
  * Creating structured User Groups
  * Writing and attaching custom policies for precise permissions

---

## 📝 Project Reflection
* **Time to Complete:** ~30 minutes
* **The Challenge:** The most challenging part was the cleanup process because I had to make sure that I didn't miss anything and leave unwanted resources running.
* **The Reward:** Managing the users was incredibly rewarding. This is a crucial skill you need not just in AWS, but across any cloud platform or modern security application.
